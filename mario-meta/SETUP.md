# SETUP — איך מחברים את מריו ל-Meta (פעם אחת)

המדריך הזה הוא ל-**ניר**. הוא עונה על השאלה "איך מריו בעצם מעלה תוכן ומושך נתוני
קמפיין מפייסבוק/אינסטגרם". יש שני מסלולים — **מסלול A (Make) מומלץ ומהיר**, ומסלול B
(API גולמי) אופציונלי לעתיד.

> ⚠️ **חוק הברזל:** גם אחרי החיבור, מריו לא מפרסם ולא נוגע בתקציב לבד. תוכן אורגני
> מתפרסם רק אחרי "אשר", וכל דבר שכרוך בכסף תמיד עוצר לאישור ידני שלך.

---

## מסלול A — דרך Make.com (מומלץ) ⭐

**החדשות הטובות: רוב העבודה כבר עשויה.** בחשבון ה-Make שלך (Niro-דיגיטל, Pro) כבר
קיימים **חיבורי Facebook פעילים** (ע"ש Nir Vaishenker), וגם Facebook Lead Ads כבר רץ
בתרחיש לידים. מריו עובד דרך התשתית הזו — בלי לבנות Meta App מאפס ובלי App Review.

### שלב 1 — לוודא שהחיבור חי והרשאות מספיקות
תגיד לראובן: **"מריו, תבדוק חיבור Meta"**. ראובן יריץ דרך ה-Make MCP בדיקת חיבורים
(`connections_list`) ויוודא שחיבור ה-Facebook קיים ולא פג תוקף. בנוסף צריך לוודא
שלחיבור יש את ההרשאות (scopes) הנכונות לפי מה שתרצה שמריו יעשה:
- **לפרסום פוסטים:** `pages_manage_posts`, `pages_read_engagement`.
- **לפרסום באינסטגרם:** `instagram_basic`, `instagram_content_publish`.
- **לקריאת נתוני קמפיינים:** `ads_read` (ולשינוי קמפיין: `ads_management`).

אם חסרה הרשאה — נכנסים ל-Make → Connections → לחיבור ה-Facebook → **Re-authorize**,
ומסמנים את ההרשאות החסרות. זה תהליך של דקה דרך הדפדפן.

### שלב 2 — תרחישי הביצוע
מריו **מאפיין** איזה תרחיש Make משמש לכל פעולה, וראובן מריץ אותו אחרי אישור:
- **פרסום פוסט** — תרחיש Make עם מודול "Facebook Pages → Create a Post" / "Instagram → Create a Post".
- **משיכת נתוני קמפיין** — מודול "Facebook Ads → Get Insights" / Lead Ads (כבר קיים).

אם תרחיש כזה עוד לא קיים, מריו יאפיין אותו ונבנה אותו פעם אחת (דרך ה-Make MCP או ידנית ב-Make).

### שלב 3 — בדיקה בטוחה לפני פוסט אמיתי
לפני הפוסט החי הראשון — בודקים על **פוסט טיוטה / קהל מצומצם**, מאשרים שהכול תקין, ורק אז מפרסמים אמיתי.

## מסלול B — API גולמי (אופציונלי, לעתיד)

רק אם בעתיד תרצה שליטה ישירה מעבר ל-Make (למשל אוטומציות מתקדמות). זה דורש להוציא
טוקן ולמלא את `META_*` ב-`.env` (המשתנים כבר מוכנים ב-`.env.example`):

```
META_APP_ID=
META_APP_SECRET=
META_ACCESS_TOKEN=        # טוקן ארוך-טווח (long-lived) של ה-Page
META_PAGE_ID=
INSTAGRAM_BUSINESS_ACCOUNT_ID=
META_AD_ACCOUNT_ID=       # act_XXXXXXXX — לקריאת/ניהול קמפיינים
```

איך משיגים: ב-[developers.facebook.com](https://developers.facebook.com) יוצרים App,
מוסיפים מוצרים (Facebook Login / Marketing API), מפיקים טוקן עם ההרשאות מהרשימה למעלה,
וממירים אותו ל-long-lived. את ה-Page/IG/Ad-Account IDs מוצאים ב-Business Suite. אחרי
מילוי `.env`, הסקיל `meta-publish` (Phase 2) יודע לקרוא ולפרסם ישירות.

> ⚠️ הקובץ `.env` לא עולה ל-git (ב-gitignore), אז הטוקנים נשארים רק אצלך במחשב.
> טוקנים של מטא פגים — צריך לרענן מדי תקופה. מסלול Make חוסך את הכאב הזה.
