# agent-eitan — הגדרת הסוכן איתן

**קובץ:** `.claude/agents/eitan.md`
**משויך ל:** איתן (מנהל האתר)
**סוג:** הגדרת סוכן (agent definition, עם frontmatter)

## מה הקובץ עושה

מגדיר את הסוכן **איתן** ואת זרימת העבודה שלו לניהול אתר WordPress דרך REST:

1. מקבל בקשה מראובן (איזה דף, מה לשנות).
2. **`test-auth`** ([[skill-wp-rest]]) — בלי חיבור עוצר ומפנה ל-`eitan/SETUP.md`.
3. מאתר את הדף (`list-pages`) ומריץ **אודיט Elementor**.
4. **סנאפשוט** (`get-page` → `eitan/snapshots/`) לפני כל עריכה — חובה.
5. עורך דרך `update-page`, מאמת שהשינוי תקין ושהטופס עובד, מתעד ב-`eitan/Memory/changes.md`.
6. מדווח לראובן.

## פרוטוקול הבטיחות (לב ההגדרה)

- ⚠️ **Elementor ↔ REST:** דף Elementor שומר תוכן ב-meta `_elementor_data`, לא ב-`content`.
  עדכון `content` עליו לא יופיע / יידרס — זה שורש ה"נדפק" של הטופס. איתן עוצר ושואל את ניר,
  וההמלצה הקבועה: להעביר את הטופס לבלוק **"Custom HTML"** / Code-Snippet יציב.
- סנאפשוט לפני כל עריכה; דף קריטי (משפך הלידים) אף פעם לא ראשון — קודם דף טיוטה.

**הערה ארכיטקטונית:** איתן לא קורא ישירות לסוכנים אחרים. מקבל מראובן ומדווח לראובן.
כשצריך טקסט לדף — ראובן מפעיל את יעל קודם, ואיתן מטמיע. כל פעולה מול האתר עוברת דרך `wp-rest`.
**כלים:** Read, Write, Edit, Bash, Glob, Grep.
