# SETUP — איך מחברים את אמציה (פעם אחת)

המדריך הזה הוא ל-**ניר**. הוא עונה על השאלה "אילו חיבורים אמציה צריך כדי לבנות ולתפעל
אוטומציות". רוב התשתית כבר קיימת — Make ו-Airtable כבר מחוברים כ-MCP.

> ⚠️ **חוק הברזל:** גם אחרי החיבור, אמציה לא בונה ולא מפעיל אוטומציה חיה לבד. כל פעולה
> חיה (יצירת/הפעלת Scenario, פרסום Flow, קריאת API שמשנה) רצה **רק אחרי "אשר"** מראובן.

---

## 1. Make.com — שכבת הביצוע המרכזית ⭐

**החדשות הטובות: ה-Make MCP כבר מחובר** (server פעיל בחשבון ה-Make של ניר). אמציה
**מאפיין** את התרחיש (אילו מודולים, אילו שדות, איזה Trigger), ו**ראובן** מריץ את פעולת
ה-Make MCP — רק אחרי "אשר".

### לבדוק שהחיבור חי
תגיד לראובן: **"אמציה, תבדוק חיבור Make"**. ראובן יריץ דרך ה-Make MCP בדיקת חיבורים
(`connections_list`) ויוודא שהחיבורים קיימים ולא פגי תוקף.

### מה אמציה עושה דרך Make
- בניית Scenarios (Triggers / Routers / Filters / Iterators / Aggregators / Data Stores).
- Webhooks נכנסים/יוצאים, מיפוי שדות, Error Handlers.
- ניהול Operations ועלויות — בונה תרחיש שלא מתרסק בעומס.

---

## 2. Airtable — ה-CRM הראשי

**Airtable MCP כבר מחובר** (server פעיל). אמציה משתמש בו לאבחון/קריאה (list bases,
get schema, list records). שינוי חי (יצירת Base/טבלה/רשומות) — **רק אחרי "אשר"**.

אם נדרש חיבור Airtable דרך Make (סנכרון דו-כיווני) — צריך Personal Access Token:
- היכנס ל-Airtable → Developer Hub → Personal Access Tokens → צור טוקן עם ה-scopes
  הנדרשים (`data.records:read/write`, `schema.bases:read/write`).
- שמור אותו ב-`.env`: `AIRTABLE_API_KEY=...` (ואם רלוונטי `AIRTABLE_BASE_ID=...`).

> כשמוקם לקוח חדש — אמציה בונה לו **Base משלו** לפי הצורך הספציפי.

---

## 3. ManyChat — בוטים ושיחות

- **הפלואו נבנה בעורך של ManyChat** (לא דרך API). אמציה מאפיין את המבנה (Trigger, שלבי
  שיחה, Conditions, Tags/Fields), וניר/אמציה בונים בעורך.
- **ההיגיון והניתוב** (חדש/ישן, סיווג ליד) נעשים ב-**Make** דרך Webhook.
- לחיבור ManyChat ל-Make צריך API Token: ManyChat → Settings → API → צור טוקן ושמור
  ב-`.env`: `MANYCHAT_API_TOKEN=...`.

---

## 4. כלי אוטומציה נוספים (לפי צורך)

Zapier / n8n / Apify / Google Apps Script — אמציה בוחר את הכלי הנכון למשימה. חיבור/טוקן
נוסף יתועד כאן וב-`.env` כשיידרש בפועל (אמציה לא חושף טוקנים בטקסט גלוי).

---

## בדיקה בטוחה לפני אוטומציה אמיתית
לפני ההפעלה החיה הראשונה — בודקים עם **Run once + נתוני דמה**, בודקים את הלוג, מאשרים
שהכול תקין, ורק אז מפעילים אמיתי. כסף/הודעות ללקוחות = תמיד אישור ידני מפורש.
