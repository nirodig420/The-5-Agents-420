## 0 · התמונה הגדולה — מה אנחנו בונים ולמה ככה

שני תרחישים נפרדים שמדברים זה עם זה דרך **כתובת Webhook אחת**:

- **תרחיש A ("אסי שולח"):** מזהה רשומה שעברה לסטטוס "נשלח לאישור" → מייצר token →
  שולח לניר מייל עם הקופי, הויזואל, ושני לינקים (אשר / דחה).
- **תרחיש B ("אסי מקבל"):** הלינק שניר לוחץ במייל הוא בעצם Webhook של B. B מזהה את ה-token,
  מוצא את הרשומה, ומעדכן סטטוס ל"מאושר" או "נדחה" + שולח מייל סיכום.

**עיקרון הזהב (קח אותו למשימה הבאה):** מפרידים את ה*לוגיקה* (Airtable + ניתוב) מה*ערוץ*
(מייל היום, וואטסאפ מחר). כשתעבור למסלול A (וואטסאפ אמיתי) — תחליף רק את מודול השליחה
ואת הטריגר של B, וכל השאר נשאר. בנייה אחת, שדרוג בלי לזרוק.

### ⚠️ הסדר קריטי: קודם B, אחר כך A

תרחיש A צריך את **כתובת ה-Webhook** כדי לבנות את הלינקים. הכתובת הזו נוצרת רק כשבונים את
מודול ה-Custom Webhook של תרחיש B. לכן:

1. **בונים B קודם** → רושמים את ה-Webhook (Run once כדי שיקבל את ה-URL) → **מעתיקים את ה-URL**.
2. **בונים A** → מדביקים את ה-URL במקום ה-placeholder `__WEBHOOK_B_ID__`.

## 8 · נקודות שראובן חייב לאמת מול Make MCP (הכי פחות בטוח)

ראה את הסיכום שאמציה החזיר. בקצרה, מה שחיפוש האינטרנט **לא** אישר במדויק ועלול להישבר בייבוא:

1. **שמות מודולי Airtable וגרסאותיהם:** `airtable:TriggerWatchRecords` (v3?),
   `airtable:ActionSearchRecords` (v3?), שדה הטריגר/formula במבנה ה-Watch. **אומת רק:**
   `airtable:ActionUpdateRecords` v3 (מ-blueprint אמיתי).
2. **מבנה ה-Router:** השתמשתי ב-`builtin:BasicRouter` + מערך `routes` עם `flow` בכל נתיב,
   ו-`filter` ברמת המודול הראשון בכל נתיב. ייתכן ש-Make מצפה למבנה filter שונה במקצת.
3. **`gateway:WebhookRespond`** ומיקומו אחרי Router — ראה הערת המיקום בסעיף 5.
4. **`google-email:ActionSendEmail`** v1 + שמות פרמטרים (`to`, `subject`, `html`,
   `contentType`, `attachments`). ייתכן `bodyType`/`content` במקום `html`.
5. **`{{uuid}}`** כפונקציית Make — לאמת שהשם נכון (לחלופין `{{token}}` או יצירת ID אחרת).
6. **מיפוי שדות בעברית** (`{{1.נושא}}`, `{{1.ויזואל (קובץ)[].url}}`) — ב-blueprint Make
   לעיתים השדות ממופים לפי field-id ולא לפי שם. אם נשבר — לבנות ידנית ולבחור מהרשימה.

**המלצה:** אם הייבוא של אחד הקבצים נכשל — אל תילחם בו. בנה את אותו תרחיש ידנית לפי סעיף 5
(שם המודולים בעברית/אנגלית בעורך יציבים יותר משמות ה-blueprint הפנימיים), ושמור Export
משלך כ-blueprint מאומת לפעם הבאה.
