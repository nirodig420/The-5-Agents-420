## ההצעה (מה / למה / השפעה)

**מה:** לחבר Property חדש של GA4 אל קונטיינר ה-GTM הקיים (`GTM-NXRL4W23`) — בלי לגעת
בקוד האתר, בלי אלמנטור, בלי theme.

**למה:** UTM שווים משהו רק אם GA4 קורא אותם. GA4 הוא הכלי שמדווח "איזה קישור/קמפיין/ערוץ
הביא ליד". בלי זה, כל השקעה בתיוג UTM של מריו יורדת לטמיון.

**השפעה צפויה:** לראשונה תמונת **מקורות תנועה והמרות אמיתית** (אורגני / סושיאל / קמפיין /
מייל), בסיס לכל החלטת תקציב ותוכן עתידית. אפס סיכון לאתר.

## צעד-אחר-צעד (למתחיל)

1. **פתיחת GA4** — נכנסים ל-analytics.google.com → Admin → Create Property →
   ממלאים שם ("NIRO") ואזור זמן → בוחרים פלטפורמת Web → מקבלים **Measurement ID**
   בפורמט `G-XXXXXXXXXX`. (חינם)
2. **כניסה ל-GTM** — tagmanager.google.com → קונטיינר `GTM-NXRL4W23`.
   (דורש שלניר תהיה גישה לחשבון ה-GTM — זה החסם היחיד לבדוק מראש.)
3. **תג חדש** — Tags → New → Tag Type: **Google Tag (GA4 Configuration)** →
   מדביקים את ה-`G-XXXX` → Trigger: **All Pages (Initialization)** → Save.
4. **Preview** — לוחצים Preview, נכנסים לאתר, מוודאים שה-GA4 נורה (רואים page_view).
5. **Publish** — לוחצים Submit/Publish בקונטיינר.

זהו. אין נגיעה בקוד ה-theme PHP, **אין נגיעה באלמנטור**, אין סנאפשוט נדרש,
אין סיכון לדרוס את הטופס.

## רמת מורכבות וסיכון

- **מורכבות:** נמוכה (15 דקות בממשק GTM).
- **סיכון לאתר:** כמעט אפס — הכל בממשק GTM הנפרד, מחוץ לאתר.
- **Standing Rule #3 (סנאפשוט/אלמנטור/theme):** **לא מופעל** — לא נוגעים בקוד האתר כלל.
- **חסם יחיד לבדוק:** גישת ניר לחשבון ה-GTM של האתר.

## אחרי GA4 — השלב הבא (מריו)

ברגע ש-GA4 פעיל ב-GTM → אפשר להתקדם לסכמת **תיוג UTM אחידה** לכל הקישורים הנכנסים
(סושיאל, מייל, קמפיינים) → הנתונים ייקלטו אוטומטית ב-GA4 → Traffic Acquisition.
