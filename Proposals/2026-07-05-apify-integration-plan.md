**מאת:** אמציה (מומחה אוטומציות ו-API)

**מטרה עסקית:** לדעת מה המתחרים של NIRO עושים (בגוגל, בבלוגים, בסושיאל) ולעזור ל-nirodigital.co.il להקדים אותם בגוגל.

## 1. מה זה Apify? (הסבר לניר, מאפס)

**Apify** היא פלטפורמת ענן לסקרייפינג — כלומר, לאיסוף אוטומטי של מידע מאתרים.
במקום לכתוב קוד סקרייפינג בעצמך, יש בה חנות של יותר מ-47,000 **Actors**.

מושגים בסיסיים (משפט אחד לכל מושג):

- **Actor** — תוכנה מוכנה שרצה בענן של Apify ועושה משימת איסוף אחת (למשל: "תביא לי את תוצאות החיפוש בגוגל למילה X"). אתה לא מתקין כלום — רק נותן קלט ומקבל פלט.
- **Run (ריצה)** — הפעלה אחת של Actor; כל ריצה עולה כסף לפי כמות התוצאות או משאבי המחשוב.
- **Dataset** — הטבלה שבה Apify שומר את התוצאות של כל ריצה (אפשר להוריד כ-JSON/CSV).
- **API Token** — "מפתח" סודי שמזהה את החשבון שלך; בעזרתו מערכות חיצוניות (כמו Claude Code או Make) מפעילות את Apify בשמך. לעולם לא שומרים אותו בטקסט גלוי.
- **MCP (Model Context Protocol)** — פרוטוקול שמאפשר ל-Claude "לדבר" ישירות עם כלים חיצוניים. ל-Apify יש שרת MCP רשמי בכתובת `https://mcp.apify.com` — אחרי חיבור, ראובן/אמציה יכולים לחפש Actors, להריץ אותם ולקרוא תוצאות ישירות מהשיחה.
- **Compute Unit (CU)** — יחידת המדידה של Apify לזמן מחשוב (1 גיגה זיכרון למשך שעה); חלק מה-Actors מתומחרים לפי CU וחלק לפי מספר תוצאות.

**למה זה מתאים למטרה שלך:** במקום לבדוק ידנית איפה אתה מדורג בגוגל ומה המתחרים מפרסמים — Actor עושה את זה אוטומטית, בעברית, מגוגל ישראל, כל שבוע, והתוצאות מוזנות ישר לזיכרון של איתן (SEO) ולדיי'גסט של חן.

## 3. ה-Actors הרלוונטיים לניר (אומתו מול הדפים הרשמיים)

### א. Google Search Scraper (apify/google-search-scraper) — הכי חשוב
- **מה עושה:** מושך את דף תוצאות החיפוש של גוגל (תוצאות אורגניות, מודעות, People Also Ask, ואפילו AI Overviews) לכל מילת מפתח.
- **גוגל ישראל בעברית — נתמך רשמית:** פרמטר `countryCode: "il"` (גוגל ישראל) + `languageCode: "he"` + `searchLanguage` לעברית. זה בדיוק מה שצריך למעקב דירוגים אמיתי של NIRO.
- **עלות:** 1.80$ לכל 1,000 דפי תוצאות (מודל pay-per-event — משלמים רק על מה שרץ). כלומר דף תוצאות אחד ≈ 0.002$.

### ב. Website Content Crawler (apify/website-content-crawler)
- **מה עושה:** נכנס לאתר/בלוג של מתחרה ומחלץ את כל הטקסט כ-Markdown נקי — מושלם כדי לראות מה מתחרים מפרסמים ולהזין את זה לניתוח.
- **עלות:** לפי CU. סדר גודל: 0.2$–5$ לכל 1,000 עמודים (זול כשהאתר פשוט, יקר כשצריך דפדפן מלא). לבלוג מתחרה טיפוסי (עשרות עמודים) — אגורות.

### ג. סושיאל מתחרים — Instagram Scraper (apify/instagram-scraper) ו-Facebook Posts Scraper
- **מה עושים:** מושכים פוסטים ציבוריים מדפי מתחרים — על מה הם כותבים, כמה מעורבות מקבלים.
- **עלות:** Instagram רשמי ≈ 2.70$ ל-1,000 תוצאות בתוכנית החינמית; Facebook — יש גרסאות מ-0.75$ עד 4$ ל-1,000 פוסטים.
- **המלצה שלי: שלב 2, לא עכשיו.** סקרייפינג של Meta הוא אזור אפור מבחינת תנאי השימוש שלהם (ראה סיכונים), ומריו כבר מכסה מתחרים בסושיאל. מתחילים מגוגל — שם הערך הכי גבוה למטרה "להיות לפניהם בגוגל".

## 4. מסלול החיבור המומלץ — Apify MCP Server

### למה MCP ולא רק Make?
- **MCP** = עבודה אינטראקטיבית: אמציה/איתן שואלים "איפה אנחנו מדורגים על X?" ומקבלים תשובה חיה בשיחה, בלי לבנות כלום מראש.
- **מודול Apify ב-Make** (קיים רשמית) = לאוטומציה מתוזמנת קבועה (הריצה השבועית). שני המסלולים משלימים — מתחילים מ-MCP (מהיר לחבר), ועוברים ל-Make כשרוצים תזמון אוטומטי מלא.

### הצעדים של ניר (ידני, חד-פעמי, ~10 דקות):

**צעד 1 — פתיחת חשבון:** נכנסים ל-`apify.com` → הרשמה (אימייל או Google). בלי כרטיס אשראי. מקבלים אוטומטית 5$ קרדיט חודשי.

**צעד 2 — למצוא את ה-API Token:** ב-Apify Console (הממשק הניהולי) → **Settings → API & Integrations** → שם מופיע ה-Token. מעתיקים אותו — ולא מדביקים אותו בשום צ'אט או קובץ גלוי, רק במקום שנגדיר.

**צעד 3 — חיבור ל-Claude Code:** פותחים טרמינל בתיקיית הפרויקט ומריצים פקודה אחת:

```
claude mcp add --transport http apify https://mcp.apify.com
```

בהתחברות הראשונה Claude יבקש אימות — מריצים `/mcp` בתוך Claude Code ובוחרים authenticate; נפתח דפדפן, מתחברים לחשבון Apify ומאשרים (OAuth — בלי להדביק Token בכלל; זו הדרך המומלצת בתיעוד הרשמי).

חלופה אם ה-OAuth לא עובד: חיבור עם ה-Token בכותרת —

```
claude mcp add --transport http apify https://mcp.apify.com --header "Authorization: Bearer <ה-Token-שלך>"
```

(מחליפים את `<ה-Token-שלך>` בערך האמיתי; לא שומרים את הפקודה עם הטוקן בקובץ.)

**צעד 4 — בדיקה:** בתוך Claude Code מריצים `/mcp` ומוודאים ש-apify מופיע כמחובר. אז ראובן יכול לבקש בדיקת דמה: "חפש Actor בשם google-search-scraper" (חיפוש = קריאה בלבד, לא עולה כסף).

**מה השרת נותן אחרי חיבור (מהתיעוד הרשמי):** חיפוש Actors בחנות, הרצת Actor (`call-actor`), קריאת תוצאות (`get-dataset-items`), ואפילו חיפוש בתיעוד של Apify. אפשר לצמצם לכלים ספציפיים עם פרמטר `?tools=` בכתובת.

> ⚠️ חוק הברזל נשאר: גם אחרי החיבור, כל ריצת Actor אמיתית (= עולה קרדיט) רצה רק אחרי "אשר".

### חלופה משלימה — מודול Apify ב-Make (לשלב 2)
ב-Make יש אינטגרציה רשמית של Apify: מודולי "Run an Actor", טריגר "Watch Actor Runs", ו-"Get Dataset Items". אימות ב-OAuth או API Token. זה המסלול לריצה השבועית האוטומטית המלאה: Schedule → Run Actor → Get Dataset Items → עדכון Airtable/קובץ → התראה. נבנה אחרי שה-MVP דרך MCP מוכיח את עצמו.

## 5. מקרה שימוש ראשון (MVP): מעקב מתחרים שבועי בגוגל ישראל

**התרחיש:** פעם בשבוע (יום ראשון בבוקר, לפני הדיי'גסט של חן והבאצ' של יעל) מריצים את Google Search Scraper על 10–15 מילות מפתח של NIRO — מגוגל ישראל, בעברית.

**דוגמאות למילות מפתח** (הרשימה הסופית תסונכרן עם `eitan-seo/Memory/keywords.md` של איתן): "אוטומציה לעסקים", "סוכן AI לעסק", "בניית צ'אטבוט לעסק", "אוטומציות שיווק", "CRM לעסק קטן", וכן הלאה.

**מה מקבלים מכל ריצה:**
1. הדירוג של nirodigital.co.il בכל מילת מפתח (ואם בכלל בעמוד הראשון).
2. מי המתחרים שמעלינו — שמות, כתובות, כותרות ותיאורים (מה עובד להם).
3. שינויים משבוע לשבוע — מי עלה, מי ירד, מתחרה חדש שנכנס.

**לאן זה זורם:**
- `eitan-seo/Memory/keywords.md` — טבלת דירוגים מתעדכנת (איתן).
- `eitan-seo/Memory/competitors.md` — מתחרים חדשים שהתגלו (איתן).
- הדיי'גסט השבועי של חן — שורת "תנועות בגוגל השבוע".

**הערכת עלות חודשית (ריאלית, מחושבת):**
- 15 מילות מפתח × דף תוצאות אחד × 4.3 שבועות ≈ **65 דפי תוצאות בחודש**.
- מחיר: 1.80$ ל-1,000 דפים → 65 דפים ≈ **0.12$ בחודש**. כן, בערך 40 אגורות.
- גם אם נוסיף Website Content Crawler על 2–3 בלוגים של מתחרים פעם בחודש (~200 עמודים ≈ 0.05$–1$) — **הכול נשאר עמוק בתוך 5$ החינמיים. עלות בפועל: 0 ש"ח.**

**שלבי הביצוע (אחרי "אשר"):**
1. ריצת בדיקה ידנית דרך ה-MCP: מילת מפתח אחת, `countryCode: il`, בדיקה שהתוצאות באמת מגוגל ישראל בעברית ושהדירוג תואם בדיקה ידנית בדפדפן.
2. הרחבה ל-15 מילות המפתח + פורמט פלט קבוע (טבלת דירוגים).
3. עדכון קבצי הזיכרון של איתן (בתיאום איתו — ראובן מתזמר).
4. שלב 2 (אופציונלי, אחרי חודש מוצלח): תרחיש Make מתוזמן שמריץ הכול אוטומטית כל יום ראשון 07:00 ושולח סיכום.

## 6. סיכונים, מגבלות ובעיות צפויות

### חוקיות וכללי משחק
- **סקרייפינג של מידע ציבורי** (תוצאות גוגל, בלוגים פתוחים) נחשב באופן רחב לגיטימי — אוספים רק מה שכל גולש רואה בלי התחברות. לא אוספים מידע אישי ולא עוקפים חומות התחברות.
- **גוגל** לא אוהבת סקרייפינג אבל זו פרקטיקה סטנדרטית בתעשיית ה-SEO (כל כלי דירוג עובד ככה); Apify מנהלים את הפרוקסי והחסימות בשבילנו — זה בדיוק הערך שלהם.
- **Facebook/Instagram** — תנאי השימוש של Meta אוסרים איסוף אוטומטי; זה קיים ונפוץ (דפים ציבוריים בלבד) אבל אזור אפור. לכן השארתי את זה לשלב 2 ובמינון נמוך, אם בכלל.

### טבלת בעיות צפויות → פתרון

| בעיה אפשרית | סימן מזהה | פתרון |
|---|---|---|
| תוצאות לא מגוגל ישראל / לא בעברית | דירוגים שלא תואמים בדיקה ידנית | לוודא `countryCode: il` + `languageCode: he`; להשוות מול חיפוש ידני בגלישה בסתר |
| הקרדיט החינמי נגמר באמצע החודש | ריצות חדשות נחסמות | ה-MVP צורך ~2% מהקרדיט — כמעט בלתי אפשרי; בכל זאת מגדירים התראת שימוש ב-Apify Console |
| Actor נכשל/תקוע | ריצה בסטטוס FAILED | בודקים את הלוג של הריצה (get-actor-log); ב-Make — Error Handler עם Retry |
| דירוג "קופץ" בין שבועות | שינויים חדים לא הגיוניים | גוגל מפרסנלת תוצאות; סקרייפינג נותן תוצאה "נקייה" יותר מדפדפן אישי — עקביות חשובה ממדידה בודדת |
| שינוי מחיר/מודל של ה-Actor | חיוב שונה מהצפוי | המחירים מוצגים בדף ה-Actor; בודקים לפני כל הרחבת שימוש |

### מגבלות טכניות שאומתו
- שאילתת גוגל מוגבלת ל-32 מילים (מגבלה של גוגל עצמה).
- בתוכנית החינמית: מקסימום 25 ריצות במקביל, 8GB RAM — הרבה מעבר לצורך שלנו.
- Website Content Crawler במצב HTTP זול לא מתמודד עם אתרים כבדי JavaScript — לאתרים כאלה עוברים למצב דפדפן (יקר פי כמה, עדיין אגורות בהיקף שלנו).
